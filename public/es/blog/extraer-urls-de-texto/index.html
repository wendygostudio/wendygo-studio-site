Cómo Extraer URLs de un Texto Online — Sin Regex, Sin Terminal

Tienes logs de una API, un volcado de un fichero de configuración o un documento lleno de enlaces mezclados con texto. Aquí tienes cómo sacar todas las URLs en segundos, en local, sin subir tus datos a ninguna parte.

26 de junio de 2026

Las URLs se esconden por todas partes en el texto técnico: incrustadas en logs de API junto a timestamps y códigos de estado, repartidas por ficheros de configuración entre claves y comentarios, enterradas en HTML copiado mezclado con texto. Leerlo todo a mano para encontrarlas es lento. Usar una regex requiere acordarse del patrón. Subirlo a una herramienta online significa poner URLs potencialmente sensibles — endpoints de APIs internas, dashboards privados, entornos de desarrollo — en el servidor de otro.

TextForge es una extensión de Chrome con una función Extraer URLs que se ejecuta completamente en tu navegador. Pegas el texto, aplicas la función, copias la lista. Nada sale de tu máquina.

Cuándo necesitas extraer URLs

Logs de API y trazas de peticiones Las respuestas de API, los logs de peticiones y las salidas de tests suelen contener URLs incrustadas junto a códigos de estado, timestamps y payloads. Cuando necesitas aislar las URLs de los endpoints para depurar o testear, extraerlas es más rápido que buscarlas a mano.
Ficheros de configuración y exportaciones Los manifiestos de Kubernetes, los ficheros de Docker Compose, los exports de variables de entorno y los .env a veces contienen URLs mezcladas con claves, rutas y comentarios. Extrae las URLs para auditar los endpoints con los que habla tu servicio sin tocar el resto de la config.
HTML copiado o scrapeado Cuando copias HTML de una página web a un editor de texto, te llevas un muro de enlaces enterrados en marcado y texto. Extrae las URLs de los href para construir una lista de todos los destinos de una página, útil para auditar enlaces o detectar enlaces rotos.
Documentación y runbooks La documentación interna, los runbooks y los informes de incidentes acumulan enlaces en el texto, en tablas y en notas al pie. Extrae el conjunto completo para ver todos los recursos referenciados en el documento sin escanear línea a línea.

Por qué las alternativas manuales se quedan cortas

MétodoLa fricción
Escanear a ojo Lento para cualquier cosa de más de una página; es fácil saltarse una URL o teclearla mal.
Regex en VS Code Requiere conocer el patrón de URL y cambiar al modo buscar-y-reemplazar.
re.findall en Python Requiere una terminal, tener Python instalado y escribir un patrón que cubra las variantes http/https/ftp.
Extractor de URLs online Tus APIs internas, URLs de desarrollo y endpoints de configuración se envían a un servidor de terceros.

Una extensión de navegador elimina toda esa fricción: un clic desde la barra de herramientas, sin terminal, y todo el procesamiento ocurre en local, en tu máquina.

Cómo extraer URLs con TextForge

  1. 1
    Instala TextForge
    Instala TextForge desde la Chrome Web Store. Después, abre el menú de Extensiones y fija el icono de TextForge a tu barra para tenerlo siempre a mano.
  2. 2
    Abre la extensión
    Haz clic en el icono de TextForge en la barra del navegador. El panel de la extensión se abre al instante — sin pestaña nueva, sin redirecciones.
  3. 3
    Pega tu texto
    Pega el bloque de texto en bruto en el área de entrada. Pueden ser logs de API, un fichero de configuración, HTML copiado, documentación o cualquier texto con URLs que necesites extraer.
  4. 4
    Aplica Extraer URLs
    Selecciona Extraer URLs en el menú de herramientas. TextForge escanea toda la entrada y devuelve cada URL que encuentra, una por línea.
  5. 5
    Copia el resultado
    La lista extraída aparece al instante. Haz clic para copiarla al portapapeles. Sin esperas, sin viajes al servidor.

Ejemplo práctico

Imagina que estás depurando y recibes este bloque de texto de un log de API:

Entrada — log y documentación mezclados
Error at 12:34:05: request to https://api.internal.example.com/v1/users failed.
See runbook at https://wiki.company.net/incidents/api-failures
Fallback endpoint: https://api-backup.example.com/v1/users (untested)
Contact: admin@example.com

Tras aplicar Extraer URLs en TextForge, la salida es:

Salida — URLs extraídas
https://api.internal.example.com/v1/users
https://wiki.company.net/incidents/api-failures
https://api-backup.example.com/v1/users

Tres URLs sacadas de un texto mezclado que contenía un email, timestamps y lenguaje natural — todos los formatos de URL extraídos, sin regex, y con los endpoints internos sensibles sin salir de tu máquina.

Otras funciones de extracción en TextForge

TextForge también puede extraer direcciones de email y direcciones IP de un bloque de texto — útil cuando los logs mezclan varios tipos de datos y necesitas aislarlos de uno en uno. Las tres funciones de extracción están incluidas en la versión gratuita.

Preguntas frecuentes

¿TextForge extrae URLs de dentro de etiquetas HTML?

Sí. Las URLs dentro de href=, src= y otros atributos HTML se detectan, igual que las URLs sueltas en el texto.

¿TextForge maneja URLs con parámetros de consulta?

Sí. La URL completa, incluyendo ruta, query string y fragmento (#), se extrae como una sola unidad.

¿Extraer URLs es gratis en TextForge?

Sí. Todas las funciones de extracción — emails, URLs, direcciones IP — están incluidas en la versión gratuita. Sin cuenta ni suscripción.

¿Qué pasa con mis URLs cuando uso TextForge?

Nada sale de tu navegador. TextForge es una extensión de Chrome que procesa el texto en local, en tu máquina. No se envía ningún dato a los servidores de Wendygo Studio ni a ningún servicio de terceros.

¿Puedo extraer las URLs de una página web que estoy viendo?

TextForge trabaja sobre el texto que pegas en su área de entrada. Para extraer los enlaces de una página, selecciona todo el texto de la página (Ctrl+A), cópialo y pégalo en TextForge. La extensión extrae entonces todas las URLs de ese texto.
