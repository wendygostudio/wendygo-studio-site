How to Take a Full Page Screenshot in Chrome

Three methods to capture the entire scrollable content of a webpage — from the quick-and-dirty DevTools trick to the one-click power-user workflow.

Chrome doesn't make full-page screenshots obvious, but there are three ways to do it — no third-party service required. Here's how each one works, when to use it, and what its limitations are.

Method 1: Chrome DevTools (No Extensions Required)

Chrome ships with a hidden full-page screenshot command built directly into DevTools. It's free, requires nothing to install, and works on any page.

Steps

  1. Open the page you want to capture.
  2. Open DevTools: press F12 — or Ctrl+Shift+I on Windows/Linux, Cmd+Option+I on Mac.
  3. Open the Command Menu: Ctrl+Shift+P (Windows/Linux) or Cmd+Shift+P (Mac).
  4. Type screenshot in the search box.
  5. Select "Capture full size screenshot".
  6. Chrome saves the PNG to your Downloads folder automatically.
Tip: If the page looks blurry on a HiDPI/Retina screen, open the Device Toolbar (Ctrl+Shift+M) and set the Device Pixel Ratio to 2 before running the capture command.

Limitations

  • Requires three separate keyboard shortcuts to access.
  • No annotation tools — you get a flat PNG, nothing more.
  • No custom filename or save location.
  • Pages with lazy-loaded images (content that loads as you scroll) may appear blank in those sections. Workaround: scroll to the bottom of the page manually before capturing so all content loads.
  • Defaults to 1× resolution — can look soft on HiDPI displays.

Best for: occasional one-off screenshots when you have no extensions installed and don't want to add any.

Method 2: Native OS Screenshot Shortcuts

OS-level shortcuts capture what's visible in the browser window — not the full scrollable page. They're fine for quick grabs of the viewport, but they stop at the fold.

Useful for UI mockups or sharing a quick view of something on screen. Not a solution for long pages.

Method 3: FrameForge Extension (Fastest, Most Reliable)

If you take screenshots regularly — for documentation, QA testing, design reviews, or content creation — the DevTools method becomes a bottleneck fast. FrameForge is a Chrome extension built specifically for this workflow.

What it adds over DevTools

  • One-click capture from the toolbar icon.
  • Custom keyboard shortcut — configure once, use everywhere.
  • Handles lazy-loaded content automatically by scrolling the page before stitching the full image together.
  • HiDPI-aware — captures at your screen's native pixel density.
  • Optional annotations — add arrows, highlights, and text directly on the screenshot before saving.
  • Custom filename patterns — e.g., {domain}-{date} auto-names each capture.
  • PNG or JPEG output with adjustable quality.
  • Copy to clipboard or download — your choice, every time.

How to use it

  1. Install FrameForge from the Chrome Web Store (free).
  2. Navigate to any page.
  3. Click the FrameForge icon in the Chrome toolbar — or press your configured keyboard shortcut.
  4. The full page is captured and presented: download it, copy it to the clipboard, or add annotations before saving.

Troubleshooting Common Issues

Screenshot cuts off at the bottom

The page is using lazy loading — images or sections load only when they enter the viewport. The DevTools method captures the DOM as-is, so those sections appear blank. Fix: scroll to the bottom of the page manually before triggering the DevTools command, or switch to FrameForge which handles this automatically.

Page renders in mobile layout

The Device Toolbar in DevTools is active (phone/tablet emulation mode). This changes the viewport width and User-Agent string, causing responsive sites to show their mobile layout. Close the Device Toolbar before capturing.

Screenshot is blurry

DevTools defaults to 1× device pixel ratio. Open the Device Toolbar (Ctrl+Shift+M), set DPR to 2, then re-run the capture. FrameForge reads your actual screen DPR automatically.

Which Method Should You Use?

Use DevTools if you need a screenshot once in a while and have no reason to install an extension. The three-shortcut dance is annoying but perfectly workable for infrequent use.

Use an OS shortcut if you only need the visible viewport — handy for quick UI screenshots.

Use FrameForge if you do this regularly, work on pages with lazy-loaded content, need consistent HiDPI output, or want annotations without opening a separate image editor.
